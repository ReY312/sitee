# Отчет по реализации и тестированию

## Что реализовано

1. Fullstack-приложение (Node.js) с формой записи:
   - ФИО;
   - СНИЛС;
   - дата и время приема.
2. Сохранение данных напрямую в Supabase PostgreSQL через RPC.
3. Защита от дублирующих заявок по СНИЛС до момента посещения.
4. Комплекс мер безопасности на backend и DB-уровне.

## Безопасность

- Сервер использует только `service_role` для RPC-запросов.
- CSRF token в cookie + заголовке.
- Проверка `Origin`.
- Ограничение размера JSON body.
- Memory rate limit по IP.
- Безопасные заголовки (CSP, HSTS, anti-clickjacking).
- Валидация и нормализация входных данных.
- Хеширование IP перед сохранением.
- RLS включен, direct-access для `anon/authenticated` отозван.

## Результаты тестирования

- Unit-тесты валидации и проверки СНИЛС.
- Интеграционные тесты API (успешная запись, дубликат СНИЛС, CSRF).
- Все тесты проходят (см. команду `npm test`).

## Исправленные проблемы

- Добавлена строгая обработка некорректного JSON.
- Добавлена защита от слишком больших payload.
- Добавлена атомарная защита от дублей на уровне SQL-индекса и функции.
